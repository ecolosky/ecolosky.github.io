About Me

I have been involved with computers for the majority of my life. At a young age I enjoyed tinkering with the insides of computers, with at first little to no success. This grew into a hobby of implementing obscure 3rd party software and constructing my own personal computers. Curiosity slowly grew the hobby into an addiction, and led to software development. The third party software I used showed weaknesses that demonstrated to me the need for improvements both in stability and usability.

I started programming in Java my junior year of high school. This robust language gave me a taste of something I craved, the ability to make something that was my own. The next big step was my entrance into Rensselaer Polytechnic Institute as a major in computer science. The college setting thoroughly stimulated my craving for programming, both in intensity and breadth of usages. These four years gave me exposure to new languages, frameworks, design patterns, and ideas that were beyond my existing knowledge.

The next step for me is software development in a professional setting. I am looking for a fast paced agile team of developers that need an entry level developer. I have experience throughout the full stack of development and the curiosity to learn new tricks of the trade. I have also been exposed to agile methodologies and version control from group work at school. If you have any questions please contact me, thank you.

Rensselaer Tour Guide App

Previous Next

Features

  • The first tab is a campus map image overlaid on a Google map to show the user's location.
  • The second tab is a QR scanner used to scan codes at each destination.
  • Finally a modal then displays the information about the building.

Front End Technologies Used

  • This cross platform app is built using Intel XDK and consists of HTML, CSS, JS, and an XML file.
  • Bootstrap is the framework for the app's view.
  • The map tab utilizes the Google maps Java Script API.
  • The QR scanner uses Apache Cordova's bar code scanner.
  • The modal is populated via a JSON structure.

Back End Technologies Used

  • The database is a non relational structure that is built with MongoDB.
  • The server uses Node.JS for processing requests and database extraction.

Heat Seeker App

Previous Next

Features

  • The radar social map features anonymously uploaded locations that will give real time analytics to the user about the quality and quantity of other users.
  • Photo/Check-in Button is where users click to take a photo of their surroundings, in social media trends users tend to like to show off the places they frequent.
  • The Feed page is where users can browse photos taken by other users within a preference set radius. This allows users to stay in touch with other people on the app.

Front End Technologies Used

  • This app is built using Adobe PhoneGap which creates a cross platform app from HTML, CSS, and JS scripts.
  • Framework 7 is the core infrastructure used in the creation of the view.
  • Google maps API was used in the construction of the radar social map. The map is then overlaid with the heat map feature to show activity.
  • The check-in feature uses the Apache Cordova plug-in to take pictures using the devices camera.

Back End Technologies Used

  • The data for Heat Seeker is stored in a MySQL database.
  • WEB2PY is used as the server responsible for populating the app with content. The server also acts as a controller to update the data model.
